## 🔐 БЕЗОПАСНОСТЬ

### Аутентификация в админке
- Проверка логина и пароля в браузере
- Хранение сессии в `sessionStorage` (удаляется при закрытии браузера)
- Простая, но надёжная защита для внутреннего использования

### Рекомендации для production
1. Переместить логин/пароль на серверную часть
2. Использовать токены (JWT) для аутентификации
3. Хранить данные на сервере вместо localStorage
4. Использовать HTTPS для защиты данных при передаче

## ✨ ОСОБЕННОСТИ РЕАЛИЗАЦИИ

### 1. localStorage вместо Server-Side
**Преимущества:**
- Быстрая загрузка (нет запросов к серверу)
- Работает оффлайн
- Просто в реализации

**Недостатки:**
- Данные теряются при очистке кэша
- Не синхронизируется между устройствами
- Лимит ~5-10 MB

### 2. Динамическая загрузка контента
**Как работает:**
1. Админка сохраняет данные в localStorage
2. Обычные страницы загружают данные из localStorage при открытии
3. CSS переменные обновляются динамически
4. HTML обновляется JavaScript'ом

### 3. Анимации при скроллинге
**Используется Intersection Observer** для:
- Определения видимости элементов
- Оптимизации производительности
- Плавных переходов между состояниями

## 📞 ВОПРОСЫ И ОТВЕТЫ

**В: Где хранятся данные?**  
О: В localStorage браузера пользователя.

**В: Это работает без интернета?**  
О: Да, после первой загрузки все работает офлайн.

**В: Как восстановить данные?**  
О: Экспортируйте JSON из админки, затем импортируйте на другом браузере.

**В: Может ли несколько человек работать в админке одновременно?**  
О: Нет, данные локальные. Для этого нужен сервер.

**В: Почему не работает видео на главной?**
